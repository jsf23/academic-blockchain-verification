# Project Charter: PDF to Interactive Mockup Pipeline

## 1. Executive Summary
**Project Name:** PDF-to-Figma automation
**Project Manager:** [Tu Nombre]
**Status:** Initialization
**Last Updated:** 2026-04-18

---

## 2. Project Goals & Vision
Este proyecto tiene como objetivo transformar documentos estáticos (PDF) en activos visuales dinámicos (Figma) para acelerar la validación de propuestas de valor.

* Extraction: Analizar y sintetizar la información primaria del PDF original.
* Visualization: Crear un puente directo entre la teoría del documento y la interfaz de usuario.
* Interactivity: Proveer una experiencia tangible mediante un prototipo navegable.

---

## 3. Core Deliverables
A continuación se detallan los productos finales del proyecto:

| Entregable | Formato | Descripción |
| :--- | :--- | :--- |
| Data Extraction Sheet | .md / .json | Resumen de la información primaria extraída del PDF. |
| Navigation Map | Diagrama | Estructura de pantallas y flujo de usuario lógico. |
| Interactive Prototype | Figma Link | Mockup funcional basado en componentes (MPC). |

---

## 4. Technical Workflow
El proceso seguirá las siguientes fases técnicas:

1. Parsing & Analysis:
   * Identificación de propuesta de valor.
   * Definición de perfiles de usuario y "pain points".

2. Figma Integration:
   * Uso de Master Project Components (MPC) para consistencia visual.
   * Mapeo de datos del PDF a componentes de UI específicos.

3. Prototyping:
   * Configuración de Smart Animate y transiciones.
   * Generación de link compartido.

---

## 5. Scope & Constraints
> [!IMPORTANT]
> In Scope: Análisis de contenido, diseño de UI interactiva, prototipado de flujo.
> Out of Scope: Programación funcional, despliegue en servidores, redacción de contenido creativo no incluido en el PDF.

---

## 6. Success Criteria
* El prototipo debe reflejar fielmente la información del PDF.
* El link de Figma debe permitir una navegación completa del "Happy Path" del usuario.
* Reducción del tiempo de interpretación manual de requerimientos.
